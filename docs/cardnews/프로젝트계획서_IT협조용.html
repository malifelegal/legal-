월간 법령 제·개정 현황 — 사내 HTML 뉴스레터 프로젝트

요청 부서 법무팀 · 문서 성격 기술 검토 참고자료(요점) · 작성일 2026. 6. 12.

1한 줄 목적

매월 법령 제·개정 현황을 시각화한 대시보드를 사내 메일로 발송하고, 메일에서 특정 법령을 클릭하면 상세 카드뉴스(HTML)가 열리도록 하는 것이 목표입니다.

2현재 상황과 개선 필요성

  • 현재: 담당자가 매월 법령 자료를 수집·수작업 가공하여 자유 형식 파일로 팀원에게 메일 발송 중
  • 한계: 형식이 매번 달라 재활용이 어렵고, 자료 간 한눈 비교·열람이 불편함
  • 목표: 입력 양식을 표준화하고, 보기 좋은 대시보드 + 카드뉴스 형태로 자동 산출·배포하는 워크플로우로 전환

3업무 흐름 (4단계)

1차 수집 자료 중 대시보드 게재 항목 선별 → 2차 카드뉴스 제작 항목 선별
→ 3차 대시보드를 아웃룩 메일 발송용 산출물로 변환 → 4차 메일 항목 클릭 시 카드뉴스 HTML 열람

4산출물 형태

  • 대시보드: 월별 제·개정 법령을 카드 형태로 정리한 HTML (통계·필터 포함)
  • 카드뉴스: 주요 법령 1건당 개정 배경·핵심 변경·실무 영향을 담은 HTML 1장
  • 메일: 대시보드를 아웃룩 호환 형태(표 기반)로 변환해 본문 발송, 각 항목은 카드뉴스로 연결되는 링크
  • 모든 자료는 정적 HTML이며 사내 데이터만 사용 (외부 서비스·계정 연동 없음)

5IT·정보보호 부서에 확인이 필요한 사항

핵심 제약: 메일(아웃룩)은 보안상 자바스크립트가 동작하지 않아, "클릭 → 열림"은 하이퍼링크로만 구현 가능합니다. 따라서 카드뉴스 파일이 놓일 '팀원 공통 접근 위치'와 링크·실행 허용 여부 확인이 반드시 필요합니다.
구분확인 요청 내용
①팀원 전체가 접근 가능한 공유 네트워크 드라이브(파일서버)가 있는지 (UNC 경로)
②정적 HTML을 올려 사내 URL(http://…)로 접근할 인트라넷 웹 공간 / SharePoint·자료실이 있는지
③아웃룩 메일 본문의 하이퍼링크 클릭으로 ①·② 위치의 파일을 여는 것이 허용되는지
④HTML 파일을 메일 첨부로 주고받는 것이 허용되는지
⑤직원 PC에서 HTML 파일을 브라우저로 여는 것이 허용되는지 (사내 표준 브라우저)

6보안 관점 — 안심 포인트

외부 통신 없음인터넷·외부 서버로 나가는 호출이 전혀 없는 사내 전용 자료
실행 코드 최소정적 HTML 중심, 매크로·실행파일(exe) 없음
민감정보 미포함공개된 법령 정보만 사용, 개인정보·대외비 없음
경량 단일 파일파일 1개(수십 KB) 단위로 관리·배포·회수 용이

7요청 사항

위 5번 ①~⑤ 항목에 대한 가부(可否)를 회신해 주시면, 사내 환경에서 허용되는 방식에 맞춰 구성하겠습니다. 권장 방식이 있으시면 함께 안내 부탁드립니다.
